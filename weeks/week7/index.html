    Functional Javascript

  • Abstraction wraps all the logic away inside the function
  • Functions should, as much as possible, only perform a single operation.
  • Functions in functional programming should simply return a different value, rather than change the value itself
  • Functions can return functions
  • Functions can have properties and methods
  • If a function takes some time to compute a return value, we can save the result in a cache property. Then if the same argument is used again later, we can return the value from the cache, rather than having to compute the result again
  • Callbacks: functions that are passed to other functions as arguments
  • Immediately Invoked Function Expression is a function that is invoked as soon as it’s defined & are a useful way of performing a task while keeping any variables wrapped up within the scope of the function.
  • Example: (function(){ var temp = “world”; console.log(“Hello ” + temp); }());
    • A recursive function is one that invokes itself until a certain condition is met.
  • Currying is a process that involves the partial application of functions.
  • Example: function multiplier(x,y) { if (y === undefined) { return function(z) { return x * z; } } else { return x * y; } }

    • Ajax

  • Ajax is a technique that allows web pages to communicate asynchronously with a server and dynamically updates web pages without reloading. This enables data to be sent and received in the background, as well as portions of a page to be updated in response to user events, while the rest of the program continues to run.
  • Example: Like button on Facebook does not re-load the entire page.
  • Ajax means that JavaScript can be used to request resources from a server on behalf of the client. The resources requested are usually JSON data or small fragments of text or HTML rather than a whole web page.
  • function request(url) { var xhr = new XMLHttpRequest(); xhr.onreadystatechange = function() { if (xhr.readyState === 4 && xhr.status === 200) { document.getElementById(“output”).innerHTML = xhr.responseText; } }
  • function send(hero) { var xhr = new XMLHttpRequest(); xhr.open(“POST”, “http://reqr.es/api/users”, true); xhr.setRequestHeader(“Content-Type”, “application/json”); xhr.onreadystatechange = function(){ if (xhr.readyState === 4 && xhr.status === 201) { console.log(xhr.responseText); } }; }}
  • send(JSON.stringify(hero));
  • FormData
  • var form = document.forms.hero; form.addEventListener(“submit”, submitHero, false);

  • JSONP or JSON with padding is a very common technique of transporting data as it allows the same-origin policy to be bypassed without using CORS.
