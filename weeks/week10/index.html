    Client-Side Form Validation

  • Ensure all required form controls are filled out, in the correct format
  • Faster than waiting for server side validation.
  • Still use server side validation for security.
  • Reasons to use validation
  • - We want to get the right data, in the right format.
  • - We want to protect our users' data.
  • - We want to protect ourselves.
  • Two kinds of client-side validation.
  • - Built-In Form Validation
    • required
    • minlength and maxlength
    • min and max
    • type: data-type: number, etc.
    • pattern: regex match
  • - JavaScript Validation
    • patternMismatch
    • tooLong
    • maxLength
    • minLength
    • type
    • Use checkValidity()

    Using Fetch

  • The Fetch API provides a JavaScript interface for accessing and manipulating parts of the HTTP pipeline, such as requests and responses
  • fetch()
  • Example: fetch('http://example.com/movies.json') .then(response => response.json()) .then(data => console.log(data));
  • For adding credentials: credentials: 'include'
  • You can use fetch for:
  • Uploading JSON data
  • Uploading a file
  • Uploading multiple files
  • Processing a text file line by line
  • Checking that the fetch was successful
  • Supplying your own request object
  • Add headers: const myHeaders = new Headers(); myHeaders.append('Content-Type', 'text/plain');
  • Response instances are returned when fetch() promises are resolved.
  • Data included in Response.status Response.statusText Response.ok
